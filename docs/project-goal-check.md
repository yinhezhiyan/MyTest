# 项目目标核查（基于协同过滤与知识图谱的个性化习题推荐系统）

## 1. 结论（是否“完美完成”）
当前项目已经具备可运行的“题库 + 做题 + 记录 + 推荐”闭环，但**尚不能称为“完美完成”**：

- 已有推荐功能，且使用了用户历史作答（错题/次数）做个性化排序。
- 目前“知识图谱”部分主要是 `knowledge_points` 字段与按章节弱项加权，尚未看到独立图结构、关系推理或图数据库能力。
- 自测文档里提到的导入接口 `/admin/question-bank/import`，在当前控制器中未找到对应映射，文档与实现存在不一致。

## 2. 已达成目标（核心能力）
- 支持推荐接口：`GET /api/recommendations`。
- 支持学生做题、随机题、答题记录、每日推荐等流程。
- 后端启动时会初始化四个学科并导入默认题库（DS/OS/CN/CO）。

## 3. 尚未完全达成（距离“完美”）
- “协同过滤”更接近启发式加权（基于用户历史分数 + 弱章节），并非标准用户-用户或物品-物品相似度模型。
- “知识图谱”没有独立实体关系建模、图谱存储、路径推理，仅体现为知识点文本字段与章节层面的规则权重。
- 文档与接口存在偏差（导入接口）。

## 4. 题库来源核查
当前各学科题库由本仓库本地 JSON 文件提供，后端在启动时自动导入数据库：

- 数据结构：`data/question-bank/ds.json`
- 操作系统：`data/question-bank/os.json`
- 计算机网络：`data/question-bank/cn.json`
- 计算机组成原理：`data/question-bank/co.json`

此外，仓库中有对应科目的 PDF 文件（位于 `docs/question-pdf/`），但在当前后端代码中未发现直接从 PDF 解析导入到数据库的逻辑，数据库导入入口仍是上述 JSON 文件。
