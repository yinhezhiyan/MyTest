# 项目分析报告

## 1. 项目整体架构
- 这是一个典型的前后端分离项目：
  - 后端：`springboot`（Spring Boot 3 + MyBatis + MySQL）
  - 前端：`vue`（Vue 3 + Vite + Element Plus）
  - 文件资源目录：`files`（用于头像等上传文件存储）
- 后端默认端口 `9090`，前端通过 `VITE_BASE_URL` 调用接口。

## 2. 后端分析（springboot）

### 2.1 技术栈与依赖
- Spring Boot `3.3.1`，Java `21`。
- MyBatis + PageHelper 做数据访问与分页。
- Hutool 用于工具能力（如文件处理）。

### 2.2 现有功能
- `WebController`：登录、修改密码、基础连通接口。
- `AdminController`：管理员 CRUD + 分页查询。
- `FileController`：本地文件上传/下载（用于头像）。
- `GlobalExceptionHandler`：统一异常包装成统一 `Result` 返回。

### 2.3 设计优点
- 结构清晰：Controller/Service/Mapper 分层明确。
- 接口风格统一：统一 `Result` 格式，便于前端接入。
- 分页能力完整：`PageHelper + PageInfo` 使用简洁。

### 2.4 主要风险与问题
1. **明文密码风险**
   - 登录与修改密码直接做明文比对，缺少密码哈希（如 BCrypt）。
2. **默认数据库凭据硬编码**
   - `application.yml` 中出现固定账号密码（`root/123456`），生产风险高。
3. **文件上传安全性不足**
   - 仅拼接原文件名与时间戳，缺少类型/大小白名单和安全校验。
4. **逻辑提示文案疑似错误**
   - 新增管理员时若用户名已存在，抛出的文案是“用户不存在”，语义不一致。
5. **CORS 配置较宽松**
   - 允许任意来源、任意头、任意方法，建议按环境收敛。
6. **实体层重复定义字段**
   - `Admin` 继承 `Account` 但又重复声明同名字段，后续维护和序列化映射容易混淆。

## 3. 前端分析（vue）

### 3.1 技术栈
- Vue 3 + Vue Router。
- Axios 封装请求拦截器。
- Element Plus 作为 UI 组件库。

### 3.2 现有能力
- 登录页（管理员角色）。
- 管理后台框架页（顶部 + 左侧菜单 + 内容区）。
- 管理员列表页：查询、分页、新增、编辑、删除、头像上传。

### 3.3 主要风险与问题
1. **本地存储 key 不一致**
   - 登录后写入 `system-user`，退出时删除的是 `code2026-user`，导致退出后残留会话数据。
2. **鉴权策略较弱**
   - 仅靠前端 localStorage 判断是否登录，缺少 token 与后端鉴权机制。
3. **构建脚本依赖环境完整性**
   - 需要可用的 `vite` 命令；如果依赖安装不完整将导致构建失败。

## 4. 运行与工程化现状
- 目前仓库内含有 `springboot/target` 与 `vue/node_modules` 产物（体量较大，不建议入库）。
- 已补充根目录 `.gitignore`，避免继续提交本地依赖和编译输出。

## 5. 建议的优化优先级（从高到低）
1. **安全优先**：
   - 密码改为 BCrypt 存储与校验；
   - 配置从 `application.yml` 移至环境变量；
   - 收敛 CORS 策略；
   - 上传文件做类型、大小、路径安全校验。
2. **一致性与稳定性**：
   - 修复前端退出登录 localStorage key；
   - 修复“用户名已存在”异常文案；
   - 去掉实体重复字段（保留父类字段即可）。
3. **工程化完善**：
   - 增加 README（启动说明、环境变量、数据库初始化）；
   - 增加基础单元测试/接口测试；
   - 增加 CI（后端编译 + 前端构建 + 基础检查）。
